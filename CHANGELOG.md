# wx_channel 更新日志

## [5.6.5] - 2026-05-15

### 原始视频与评论导出补齐

- 补齐“原始视频”下载链路，改为优先走 Gopeed 引擎下载，不再依赖前端直下路径。
- 评论列表采集与保存改为对齐 `wx_channels_download`，输出精简后的导出格式，不再保留冗余原始 JSON 字段。
- 新增并收录 `SunnyNet v1.0.3` 到 `pkg` 模块，便于开发者直接构建和协作。

### 下载命名调整

- 下载文件名默认不再附带视频 ID，改为以标题命名。
- 保留“文件名附带视频 ID”设置项，可按需重新开启。
- 同标题文件会自动追加 `(1)`、`(2)` 等序号，避免覆盖已有文件。

## [5.6.2] - 2026-04-26

### 文档与交付整理

- 清理已下线的视频号搜索页面相关代码与文档说明。
- 保留 API 搜索能力，不影响控制台和接口侧搜索调用。
- 调整仓库忽略规则，允许 `web/` 目录相关站点资源纳入版本管理。

## [5.6.1] - 2026-03-16

### 补丁修复

- 修复 `Home` 路径下分享直达视频、个人页跳转视频等“详情页模式”被误判为普通首页，导致下载按钮灰色不可用的问题。
- 修复 `Home` 路径下详情页模式点击“采集评论”提示“评论采集功能尚未就绪”的问题，补齐评论采集脚本注入。
- 优化 `Home` 页面直达视频链接识别，兼容 `oid/nid`、`context_id`、`fromSubPage=profile` 等不同 URL 形态。

## [5.6.0] - 2026-03-15

### 重点更新

- 适配微信视频号新版页面结构，重点修复 `Home / Feed / Profile / 赞和收藏(account/like)` 页面。
- 重构本地 WebSocket 与 Hub 设备选择逻辑，解决新版页面下“连上但不可用”导致的搜索和详情调用失败问题。
- 对标雷达默认改为关闭，并支持在本地控制台设置页中即时启停。

### 页面适配与稳定性修复

#### 首页 Home
- 适配新版顶部四标签结构：`直播 / 关注 / 朋友 / 推荐`
- 移除旧版左侧 12 模块相关残留逻辑
- 修复首页刷新卡在“加载中”、首屏空白等问题
- 改为更保守的首页数据同步方式，避免旧版 bundle 重写破坏新版页面初始化
- 首页切换视频后，重新输出当前视频详细信息，行为与 `feed` 页保持一致

#### Feed 页
- 适配新版 `feed` 页面头部与视频切换结构
- 修复切换上下一个视频后，下载仍指向上一条视频的问题
- 修复评论采集入口，支持先自动打开评论侧栏再开始采集
- 评论采集改为兼容新版侧栏 DOM 结构，支持从页面 DOM 直接解析评论与回复
- 优化评论采集进度提示与自动滚动逻辑，优先滚动评论区而非主视频区域
- 修复评论保存文件命名，使其与视频命名风格统一
- 后续版本已退役上述评论采集链路，改为统一的评论列表 API 获取方式

#### Profile / 赞和收藏
- 适配新版 `profile` 页面资料区按钮布局
- 批量下载按钮改为跟随新版原生操作区位置
- 修复 `account/like` 页面空白问题
- 支持“赞和收藏”页面按当前子标签单独显示批量下载内容，不再混合全部数据
- 修复滚动加载更多后批量下载数量不更新的问题

### WebSocket 与 Hub 联动增强

- 本地 `/ws/api` 增加客户端状态上报，包含：
  - 当前页面路径
  - 当前 URL
  - API 是否就绪
  - 支持的方法集合
- Hub 端记录并展示设备能力：
  - 搜索能力
  - 视频列表能力
  - 视频详情能力
  - ready 页面数量
- Hub 搜索、视频列表、视频详情调用改为优先选择真正具备对应能力的设备
- `/api/status` 返回更完整的客户端能力列表，便于排查“已连接但不可用”的问题
- Hub 前端 Dashboard / Search 页面增加能力状态展示，并优化“自动选择设备”逻辑

### 雷达与控制台

- 新增 `radar_enabled` 配置项，默认值为 `false`
- 程序启动时仅在显式启用时启动雷达服务
- 本地 Web 控制台“设置”页新增“启用对标雷达”开关
- 雷达开关支持热切换，保存后立即启停，无需重启程序

### 兼容性说明

- 为保证新版视频号页面稳定，已停用部分旧版首页 JS 源码重写逻辑
- 新版页面优先采用运行时状态与页面结构识别，不再依赖高风险的旧 bundle 硬编码替换
