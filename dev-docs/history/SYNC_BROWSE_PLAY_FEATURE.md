# 浏览记录播放功能

## 功能说明

在同步页面的浏览记录列表中，现在可以点击播放按钮直接跳转到视频详情页并播放视频。

**特别说明**: 由于浏览记录中已包含完整的视频信息（包括加密视频 URL 和解密密钥），因此可以直接播放，无需再次调用客户端 API。

## 使用方法

### 1. 进入同步页面

访问 Hub Server 的同步页面：`http://localhost:8080/sync`

### 2. 查看浏览记录

1. 在设备列表中，找到要查看的设备
2. 点击该设备行的"浏览记录"按钮（眼睛图标）
3. 弹出浏览记录对话框

### 3. 播放视频

在浏览记录对话框中：
1. 找到想要播放的视频
2. 点击该视频行最右侧的"播放"按钮（播放图标）
3. 自动跳转到视频详情页
4. 视频会自动开始播放

## 功能特性

### 浏览记录列表

显示以下信息：
- **封面图片** - 视频缩略图
- **标题和作者** - 视频标题和作者名称
- **时长** - 视频时长（格式化显示）
- **分辨率** - 视频分辨率（如 1920x1080）
- **互动数据** - 点赞数、评论数、收藏数
- **浏览时间** - 用户浏览该视频的时间
- **操作按钮** - 播放按钮（新增）

### 播放按钮

- **图标**: 播放图标（▶）
- **颜色**: 绿色
- **提示**: 鼠标悬停显示"查看详情并播放"
- **功能**: 点击后跳转到视频详情页

### 跳转逻辑

1. 从浏览记录中获取完整的视频信息
2. 通过 Vue Router 的 `state` 传递视频数据到详情页
3. 视频详情页检测到来自浏览记录的数据后，直接使用这些数据
4. 无需调用客户端 API，直接播放视频

**数据传递**:
```javascript
router.push({
  path: `/video/${record.id}`,
  query: { from: 'browse_history' },
  state: {
    browseRecord: {
      id, title, author, cover_url,
      video_url, decrypt_key, duration,
      resolution, like_count, comment_count, fav_count
    }
  }
})
```

### 错误处理

如果视频 ID 不存在：
- 显示警告提示："视频 ID 不存在，无法跳转"
- 不执行跳转操作

## 技术实现

### 前端修改

#### 1. 添加操作列

在浏览记录表格中添加了新的操作列：

```vue
<Column header="操作" style="width: 100px">
  <template #body="{ data }">
    <Button 
      icon="pi pi-play" 
      text 
      rounded 
      severity="success" 
      size="small" 
      @click="goToVideoDetail(data)" 
      v-tooltip="'查看详情并播放'" 
    />
  </template>
</Column>
```

#### 2. 导入 Vue Router

```javascript
import { useRouter } from 'vue-router'
const router = useRouter()
```

#### 3. 实现跳转函数

```javascript
const goToVideoDetail = (record) => {
  if (record.id) {
    router.push(`/video/${record.id}`)
  } else {
    toast.add({ 
      severity: 'warn', 
      summary: '提示', 
      detail: '视频 ID 不存在，无法跳转', 
      life: 3000 
    })
  }
}
```

### 数据流程

1. **浏览记录同步**
   - 客户端通过 WebSocket 推送浏览记录到 Hub Server
   - 浏览记录包含视频 ID、标题、作者、封面等信息

2. **记录查询**
   - 前端调用 `/api/sync/browse` API 获取浏览记录
   - 支持分页查询（每页 20 条）

3. **视频播放**
   - 点击播放按钮后跳转到 `/video/{id}` 路由
   - 视频详情页加载视频信息并播放

## API 接口

### 获取浏览记录

```http
GET /api/sync/browse
Authorization: Bearer <token>

Query Parameters:
- machine_id: 设备 ID
- page: 页码（从 1 开始）
- page_size: 每页记录数（默认 20）
```

## 注意事项

### 1. 视频可用性

- 浏览记录中的视频可能已被删除或下架
- 跳转后可能显示"视频不存在"错误
- 建议在视频详情页添加错误处理

### 2. 权限控制

- 确保用户有权限访问视频详情页
- 某些视频可能需要特定权限才能播放

### 3. 性能考虑

- 浏览记录列表支持分页，避免一次加载过多数据
- 封面图片使用懒加载，提升页面性能

### 4. 数据同步

- 浏览记录由客户端自动推送
- 可能存在延迟，最新的浏览记录可能需要等待同步

## 故障排除

### 问题 1: 点击播放按钮没有反应

**可能原因**:
- 视频 ID 为空
- 路由配置错误
- JavaScript 错误

**解决方法**:
1. 打开浏览器控制台查看错误信息
2. 检查浏览记录数据是否包含 `id` 字段
3. 确认路由配置正确

### 问题 2: 跳转后显示 404

**可能原因**:
- 视频详情页路由不存在
- 视频 ID 格式错误

**解决方法**:
1. 检查路由配置文件
2. 确认视频 ID 格式正确
3. 检查视频是否存在

### 问题 3: 视频无法播放

**可能原因**:
- 视频已被删除
- 解密密钥失效
- 网络问题

**解决方法**:
1. 检查视频是否仍然存在
2. 尝试重新获取视频信息
3. 检查网络连接
